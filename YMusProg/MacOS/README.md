# YMus — macOS

Порт десктоп-программы YMus под macOS (Apple Silicon: M1–M4 и новее).
Тот же интерфейс и функционал, что в Windows-версии: детект Chromium-браузеров,
автоматическая установка/обновление расширения, автозапуск, телеметрия.

## Структура

```
YMusProg/MacOS/QtApp/
  main.py            — точка входа (Qt/QML, трей, single-instance)
  backend.py         — логика: браузеры, скачивание расширения, настройки
  mac_automation.py  — автоматизация браузеров через AppleScript (System Events)
  guard.py           — лёгкий анти-дебаг (sysctl P_TRACED)
  build_mac.sh       — локальная сборка .app + .dmg (запускать НА Mac)
```

Интерфейс (QML) и иконки НЕ дублируются — при сборке берутся из
`YMusProg/Windows/QtApp/qml` и `.../assets` (они кроссплатформенные).

## Как собрать .dmg БЕЗ Mac (рекомендуется)

Сборка идёт на облачном Apple Silicon через GitHub Actions:

1. Залить репозиторий на GitHub (см. основную инструкцию).
2. Вкладка **Actions** → workflow **«Build macOS (.dmg)»** → **Run workflow**.
3. Через ~5–10 минут скачать артефакт **YMus-macos** (внутри `YMus.dmg`).
4. Отдать `.dmg` тестировщику.

Файл workflow: `.github/workflows/macos.yml`.

## Как собрать на самом Mac

```bash
bash YMusProg/MacOS/QtApp/build_mac.sh
# результат: ./YMus.dmg
```

## Запуск у тестировщика

1. Открыть `YMus.dmg`, перетащить **YMus** в «Программы».
2. Первый запуск: **правый клик по YMus → «Открыть» → «Открыть»**
   (сборка неподписанная, Gatekeeper иначе блокирует).
3. Для автоустановки расширения выдать разрешение:
   **Системные настройки → Конфиденциальность и безопасность →
   Универсальный доступ (Accessibility)** → включить **YMus**.

## Что требует проверки на реальном Mac

- Автоклик внутри `chrome://extensions` (поиск кнопок через AX-дерево).
  Названия кнопок/тумблеров могут отличаться по версии браузера и языку —
  при необходимости расширить списки в `mac_automation.py` / `backend.py`.
- Корректность имён процессов браузеров для System Events.

## Подпись (опционально, позже)
