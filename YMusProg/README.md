# YMus Desktop

YMus Desktop - приложение для установки, обновления и управления расширением
YMus в браузерах пользователя.

## Структура

- `Windows/` - техническое задание и материалы для первой версии под Windows.
- `MacOS/` - задел под последующую версию для macOS.

## Ключевое ограничение браузеров

Приложение может автоматически:

- определить установленные браузеры;
- получить с сервера информацию о последнем релизе;
- скачать подходящий пакет расширения;
- проверить подпись и контрольную сумму;
- подготовить расширение к установке или обновлению;
- применить официальные browser policy в управляемом режиме.

Обычная программа не может незаметно установить произвольное расширение во
все современные браузеры. Chrome, Edge, Firefox и Yandex Browser специально
ограничивают такие сценарии. Поэтому YMus Desktop поддерживает три режима:

1. **Через магазин** - приложение открывает официальную страницу расширения,
   пользователь подтверждает установку.
2. **Управляемая установка** - приложение с согласия пользователя применяет
   официальные политики браузера; может потребоваться UAC и управляемая среда.
3. **Режим разработчика** - приложение скачивает расширение в постоянную папку
   и проводит пользователя через одноразовое подключение распакованной версии.

Для YMus основной локальный сценарий - verified unpacked install: приложение
скачивает ZIP, проверяет подпись и хэш, распаковывает расширение в стабильный
путь и помогает пользователю подключить эту папку в браузере. Полностью скрытое
включение developer mode и загрузка unpacked-расширения не закладываются как
публичный MVP, потому что браузеры специально требуют пользовательского
подтверждения таких действий.

Секреты сервера не должны попадать в приложение. Клиент хранит только публичный
ключ проверки релизов и публичные endpoint-адреса. Приватные ключи, токены
хранилища и подпись релизов остаются на сервере/CI.

Подробное ТЗ первой версии находится в
[`Windows/TZ.md`](Windows/TZ.md).
